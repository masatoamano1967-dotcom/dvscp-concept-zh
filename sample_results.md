# DVSCP — 压缩还原实验示例

以下为针对英语、日语与中文文本的压缩实验结果。  
中间压缩数据包格式未予公开（专利申请中）。  
压缩率基于规则引擎的输出结果进行计算。

## 中文实验

### 关于中文分词

中文书写中词语之间没有空格分隔，因此分词方式对压缩率有显著影响。  
本实验分别测试了两种方式：

| 分词方式 | 说明 | 平均压缩率 |
|---|---|---|
| **字符级**（逐字处理） | 无需外部库，每个汉字独立处理 | **53.0%** |
| **词语级**（jieba 分词） | 使用 jieba 库进行中文分词 | **74.9%** |

词语级分词的压缩率与英语（77.0%）、日语（77–85%）接近，  
这进一步印证了**语义惯性是与语言无关的普遍原理**。

### 示例 1：叙事文——意志表达

**原始文本（238 bytes）**

```
马库斯缓缓转过身去。"不，这是我必须独自完成的事。我不会向任何人求助。"
他的眼神中没有丝毫犹豫。艾琳娜听到这些，什么也说不出口。
她只能望着他的背影，目送他离去。
```

| 方式 | 压缩后 | 压缩率 |
|---|---|---|
| 字符级 | 118 bytes | 50.4% |
| **词语级（jieba）** | **63 bytes** | **73.5%** |

**还原文本（Claude 生成）**

```
马库斯慢慢地转过身。"不，这件事必须由我独自完成。我不会请求任何人的帮助。"
他的目光中没有一丝迟疑。艾琳娜听后，无言以对。
她只能默默注视着他的背影，看着他渐渐远去。
```

**质量评估**
- 核心意图（马库斯的决心、艾琳娜的无助）：✅ 完整保留
- 情感基调：✅ 完整保留
- 变化：副词出现同义替换——对语义无影响

### 示例 2：技术说明文

**原始文本（231 bytes）**

```
本系统专为低带宽环境设计。
它实现了传统通信方式无法达到的数据压缩水平，
但还原后的内容并非原文的完整复现——
它保证的是语义等价性，而非字节层面的完全一致。
```

| 方式 | 压缩后 | 压缩率 |
|---|---|---|
| 字符级 | 112 bytes | 51.5% |
| **词语级（jieba）** | **57 bytes** | **75.3%** |

**还原文本（GPT-4o 生成）**

```
本系统针对带宽极为有限的环境而设计。
其数据压缩能力远超传统通信手段所能实现的水平，
但还原结果并非原始内容的精确复制——
它所保证的是语义层面的等价，而非逐字节的完全吻合。
```

**质量评估**
- 逻辑内容：✅ 完整保留
- 关键限定条件（"并非完整复现"）：✅ 保留
- 语体：略偏正式——适合技术文档语境

### 示例 3：否定与转折

**原始文本（173 bytes）**

```
他没有拒绝。然而，他也没有同意。
沉默，就是他的答案。这不是冷漠——而是一个
他早在任何人开口询问之前便已做出的决定。
```

| 方式 | 压缩后 | 压缩率 |
|---|---|---|
| 字符级 | 81 bytes | 53.2% |
| **词语级（jieba）** | **61 bytes** | **64.7%** |

**还原文本（Gemini 生成）**

```
他没有说不。但他也没有答应。
沉默便是他的回答。这不是漠然——而是一个
他早在有人开口之前便已在心中做出的决断。
```

**质量评估**
- 双重否定结构：✅ 完整保留（最关键的测试项）
- 转折对比（然而 / 但）：✅ 保留
- 情感细微差异：✅ 保留

### 示例 4：技术描述——较长

**原始文本（372 bytes）**

```
该协议将发送方与接收方的职责完全分离。
发送方仅使用确定性规则提取语义核心——编码过程中不涉及任何人工智能。
而接收方则依赖大型语言模型，从最小化的表示中重建原始消息。
这一设计意味着，随着AI模型的持续进步，
还原质量将自动提升，无需对编码器做任何修改。
```

| 方式 | 压缩後 | 压缩率 |
|---|---|---|
| 字符级 | 166 bytes | 55.4% |
| **词语级（jieba）** | **74 bytes** | **80.1%** |

**还原文本（Claude 生成）**

```
该协议在编码端与解码端之间划定了清晰的职责边界。
编码完全依赖固定的确定性逻辑——发送端无需任何AI介入。
解码则由大型语言模型承担，负责从紧凑的语义骨架中重建完整消息。
这一架构的关键推论是：底层AI的进步将自动提升还原保真度，
编码器本身无需做任何改动。
```

**质量评估**
- 职责分离概念：✅ 完整保留
- "编码端无需AI"：✅ 保留（关键区别）
- 自动提升特性：✅ 保留

## 三语言对比汇总

| 语言 | 分词方式 | 平均压缩率 |
|---|---|---|
| 日语 | 形态素分析（MeCab） | **77–85%** |
| 英语 | 规则分词 | **77.0%** |
| 中文（词语级） | jieba 分词 | **74.9%** |
| 中文（字符级） | 逐字处理 | 53.0% |

**结论：** 使用适当分词工具后，三种语言的压缩率趋于一致（约 75–85%），  
表明**语义惯性是一种与语言无关的普遍原理**，适当的分词处理是发挥这一原理的前提条件。
